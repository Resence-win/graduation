# 校园一卡通系统数据库设计（OpenGauss）

## 一、数据库概述

本系统数据库基于 **OpenGauss** 构建，用于实现校园一卡通系统中用户管理、校园卡管理、账户管理、消费记录、充值记录以及系统日志等功能的数据存储与管理。

数据库共设计 **11 张核心数据表**：

| 表名              | 说明      |
| --------------- | ------- |
| student         | 学生信息表   |
| teacher         | 教师信息表   |
| admin_user      | 管理员表    |
| campus_card     | 校园卡表    |
| account         | 账户表     |
| merchant        | 商户表     |
| merchant_type   | 商户类型表   |
| consume_record  | 消费记录表   |
| recharge_record | 充值记录表   |
| loss_record     | 挂失记录表   |
| operation_log   | 系统操作日志表 |

---

# 二、数据库实体关系

系统主要实体及关系如下：

```
学生 / 教师
      │
      │
   campus_card
      │
      │
     account
    /      \
consume   recharge
   │
merchant
   │
merchant_type

campus_card
     │
loss_record

admin_user
     │
operation_log
```

关系说明：

* 学生 / 教师 与校园卡：1 对 1
* 校园卡 与账户：1 对 1
* 账户 与消费记录：1 对多
* 账户 与充值记录：1 对多
* 商户 与消费记录：1 对多
* 校园卡 与挂失记录：1 对多
* 管理员 与操作日志：1 对多

# 三、数据表设计

## 3.1 学生表 student

```sql
CREATE TABLE student (
    id BIGSERIAL PRIMARY KEY,
    student_no VARCHAR(20) UNIQUE NOT NULL,
    name VARCHAR(50) NOT NULL,
    gender VARCHAR(10),
    college VARCHAR(100),
    major VARCHAR(100),
    class_name VARCHAR(50),
    phone VARCHAR(20),
    status INT DEFAULT 1,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

字段说明：

| 字段         | 说明 |
| ---------- | -- |
| id         | 主键 |
| student_no | 学号 |
| name       | 姓名 |
| gender     | 性别 |
| college    | 学院 |
| major      | 专业 |
| class_name | 班级 |
| phone      | 电话 |
| status     | 状态 |

---

## 3.2 教师表 teacher

```sql
CREATE TABLE teacher (
    id BIGSERIAL PRIMARY KEY,
    teacher_no VARCHAR(20) UNIQUE NOT NULL,
    name VARCHAR(50) NOT NULL,
    gender VARCHAR(10),
    department VARCHAR(100),
    phone VARCHAR(20),
    status INT DEFAULT 1,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

## 3.3 管理员表 admin_user

```sql
CREATE TABLE admin_user (
    id BIGSERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(100) NOT NULL,
    name VARCHAR(50),
    role VARCHAR(20),
    status INT DEFAULT 1,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

## 3.4 校园卡表 campus_card

```sql
CREATE TABLE campus_card (
    id BIGSERIAL PRIMARY KEY,
    card_no VARCHAR(50) UNIQUE NOT NULL,
    user_id BIGINT NOT NULL,
    user_type VARCHAR(20) NOT NULL,
    status INT DEFAULT 1,
    issue_date DATE,
    expire_date DATE,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

字段说明：

| 字段        | 说明                    |
| --------- | --------------------- |
| card_no   | 校园卡卡号                 |
| user_id   | 用户ID                  |
| user_type | 用户类型(student/teacher) |
| status    | 卡状态                   |

卡状态：

```
0 已注销
1 正常
2 挂失
```

---

## 3.5 账户表 account

```sql
CREATE TABLE account (
    id BIGSERIAL PRIMARY KEY,
    card_id BIGINT NOT NULL,
    balance NUMERIC(10,2) DEFAULT 0,
    status INT DEFAULT 1,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_account_card
        FOREIGN KEY(card_id) REFERENCES campus_card(id)
);
```

---

## 3.6 商户类型表 merchant_type

```sql
CREATE TABLE merchant_type (
    id BIGSERIAL PRIMARY KEY,
    type_name VARCHAR(50) NOT NULL,
    description VARCHAR(200)
);
```

示例类型：

```
食堂
超市
咖啡店
打印店
```

---

## 3.7 商户表 merchant

```sql
CREATE TABLE merchant (
    id BIGSERIAL PRIMARY KEY,
    merchant_name VARCHAR(100) NOT NULL,
    type_id BIGINT,
    location VARCHAR(200),
    logo VARCHAR(255),
    status INT DEFAULT 1,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_merchant_type
        FOREIGN KEY(type_id) REFERENCES merchant_type(id)
);
```

---

## 3.8 消费记录表 consume_record

```sql
CREATE TABLE consume_record (
    id BIGSERIAL PRIMARY KEY,
    account_id BIGINT NOT NULL,
    merchant_id BIGINT NOT NULL,
    amount NUMERIC(10,2) NOT NULL,
    consume_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    remark VARCHAR(200),
    CONSTRAINT fk_consume_account
        FOREIGN KEY(account_id) REFERENCES account(id),
    CONSTRAINT fk_consume_merchant
        FOREIGN KEY(merchant_id) REFERENCES merchant(id)
);
```

该表用于记录用户在校园中的所有消费行为，是系统最核心的数据表。

---

## 3.9 充值记录表 recharge_record

```sql
CREATE TABLE recharge_record (
    id BIGSERIAL PRIMARY KEY,
    account_id BIGINT NOT NULL,
    amount NUMERIC(10,2) NOT NULL,
    recharge_type VARCHAR(20),
    recharge_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    operator_id BIGINT,
    CONSTRAINT fk_recharge_account
        FOREIGN KEY(account_id) REFERENCES account(id)
);
```

充值方式示例：

```
现金
微信
支付宝
银行卡
```

---

## 3.10 挂失记录表 loss_record

```sql
CREATE TABLE loss_record (
    id BIGSERIAL PRIMARY KEY,
    card_id BIGINT NOT NULL,
    loss_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status INT DEFAULT 1,
    operator_id BIGINT,
    CONSTRAINT fk_loss_card
        FOREIGN KEY(card_id) REFERENCES campus_card(id)
);
```

---

## 3.11 系统日志表 operation_log

```sql
CREATE TABLE operation_log (
    id BIGSERIAL PRIMARY KEY,
    admin_id BIGINT,
    operation VARCHAR(200),
    operation_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    ip_address VARCHAR(50)
);
```

# 四、数据库索引设计

为了提高系统查询效率，对高频查询字段建立索引：

```sql
CREATE INDEX idx_card_no ON campus_card(card_no);

CREATE INDEX idx_account_card_id ON account(card_id);

CREATE INDEX idx_consume_account_id ON consume_record(account_id);

CREATE INDEX idx_consume_merchant_id ON consume_record(merchant_id);

CREATE INDEX idx_consume_time ON consume_record(consume_time);
```

# 五、数据库优化设计

## 5.1 索引优化

为校园卡号、账户ID、商户ID以及消费时间等字段建立索引，提高查询效率。

## 5.2 分区表优化

消费记录数据量较大，可以按时间进行分区：

```
按月份分区
按年份分区
```

这样可以减少大数据量查询时的扫描范围，提高系统性能。

## 5.3 SQL查询优化

在系统开发过程中，通过合理设计查询语句、减少全表扫描以及合理使用 JOIN 语句，提高数据库查询效率。

# 六、数据库设计总结

本系统共设计 **11 张核心数据表**，通过主键与外键建立实体之间的关联关系，实现校园一卡通系统中用户、校园卡、账户、消费记录以及充值记录等数据的统一管理。

数据库设计具有以下特点：

1. 结构清晰，实体关系明确
2. 支持校园卡生命周期管理（开卡、挂失、补卡等）
3. 支持消费与充值数据记录
4. 支持商户管理与消费统计分析
5. 通过索引与分区表设计提高数据库性能

该数据库结构能够满足校园一卡通系统的业务需求，并为后续系统开发与性能优化提供良好的基础。
